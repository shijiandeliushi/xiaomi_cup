# CyberDog 仿真中插入 D435 深度相机（预留位）全过程记录

## 1. 目标
在现有 CyberDog 仿真链路中，按 RGB 相机同样方法，将 D435 相机挂到预留安装位，并实现：

1. Gazebo 内可发布 D435 彩色/深度图像话题
2. 启动脚本自动显示 D435 独立窗口
3. D435 视角下俯用于观察地面
4. RGB 相机保持原前视，不受 D435 影响

## 3. 接入方法（按链路）

### 3.1 `robot.xacro`：确认预留安装位并调整朝向
- 使用已有预留位：`D435_camera_link` / `D435_camera_joint`
- 保持安装坐标不变，调整俯仰角用于看地面：
  - `xyz="271.994e-3 25e-3 114.912e-3"`
  - `rpy="0 1.00 0"`（当前版本，明显下俯）

说明：只改 `D435_camera_joint`，不会影响 RGB 相机安装位。

### 3.2 `gazebo.xacro`：挂载 D435 传感器插件
在 `<xacro:if value="$(arg USE_D435_CAMERA)">` 条件块下，挂载到 `D435_camera_link`：

1. 彩色相机（camera）
   - 话题映射到 `d435/color/image_raw`
2. 深度相机（depth）
   - 话题映射到 `d435/depth/image_raw`

同时保留 Gazebo 默认命名话题（如 `/d435_depth_sensor/image_raw`），用于兼容排查。

### 3.3 `race_gazebo.launch.py`：增加参数透传
- 声明 `use_d435_camera` 启动参数
- 在 `xacro.process_file(..., mappings=...)` 中传入：
  - `'USE_D435_CAMERA': use_d435_camera`

### 3.4 `launchgazebo.sh`：默认启用 D435
在启动命令中增加：
- `use_d435_camera:=true`

确保走 `launchsim.py` 时也会自动带上 D435。

### 3.5 `launchcamera.sh`：显示与话题自适配
该脚本增加了以下机制：

1. 自动探测 RGB 原始话题（优先 RGB，不优先 D435）
2. 单独探测 D435 彩色/深度话题（优先有发布者）
3. D435 窗口优先显示彩色流（正常色彩）
4. 若彩色不可用，回退显示深度流（灰度归一化）
5. 保留 QoS 适配中继，避免显示工具订阅不兼容

## 4. 本次遇到的典型问题与处理

### 4.1 `ros2 topic hz /d435/color/image_raw` 无输出
定位结果：该话题存在但 `Publisher count: 0`，实际发布在：
- `/d435_camera_sensor/image_raw`
- `/d435_depth_sensor/image_raw`

处理：显示脚本改为“按发布者自动选话题”，不硬编码单一路径。

### 4.2 `ros2 topic hz` 参数不兼容
在当前 Galactic 环境中：
- `ros2 topic hz` 不支持 `--qos-reliability`
- `ros2 topic echo` 不支持 `--once`

处理：改用基础命令与 `topic info -v` 判断 QoS/发布者。

### 4.3 D435 画面黑屏或无响应
常见根因：
1. 订阅了“有 topic 但无发布者”的话题
2. QoS 不匹配
3. GUI 刷新机制问题

处理：
- 统一保存订阅句柄
- 用定时器 `cv2.waitKey(1)` 刷新窗口
- 优先订阅有发布者话题

### 4.4 D435 与 RGB 画面几乎一致
原因：展示的是 D435 彩色流前视，且俯角不够/符号方向与预期相反。

处理：
- 增大 D435 安装位俯仰角
- 最终用 `rpy="0 1.00 0"` 让 D435 下看地面
- 同时把 RGB 原始窗口固定优先 RGB 话题，避免“两个窗口都看地面”
