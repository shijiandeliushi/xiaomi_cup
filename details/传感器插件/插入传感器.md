# 从 `launchsim.py` 启动 Gazebo 的运行路径（含 RGB 相机参数流）

## 1. 启动入口
你执行：

```bash
python3 src/cyberdog_simulator/cyberdog_gazebo/script/launchsim.py
```

`launchsim.py` 会按顺序启动 3 个脚本：
1. `launchgazebo.sh`（Gazebo 仿真）
2. `launchvisual.sh`（可视化）
3. `launchcontrol.sh`（控制）

其中第 1 步启动后会等待 5 秒，再启动后两步。

---

## 2. 调用链路（逐层）

### 第 0 层：`launchsim.py`
- 固定工程根目录：`/home/cyberdog_sim`
- 优先用 `gnome-terminal` 分终端启动；无图形环境就后台 `Popen`
- 关键脚本列表：
  - `src/cyberdog_simulator/cyberdog_gazebo/script/launchgazebo.sh`
  - `src/cyberdog_simulator/cyberdog_gazebo/script/launchvisual.sh`
  - `src/cyberdog_simulator/cyberdog_gazebo/script/launchcontrol.sh`

### 第 1 层：`launchgazebo.sh`
- `source /opt/ros/galactic/setup.bash`
- `source install/setup.bash`
- 实际执行：

```bash
python3 src/cyberdog_simulator/cyberdog_gazebo/script/gazebolauncher.py \
  ros2 launch cyberdog_gazebo race_gazebo.launch.py
```

### 第 2 层：`gazebolauncher.py`
- 只是包装器：接收后续命令并执行
- `Ctrl+C` 时会 `killall gzclient/gzserver`

### 第 3 层：`race_gazebo.launch.py`
主要做 3 件事：
1. 启动 `gzserver.launch.py`
2. （非 headless 时）启动 `gzclient.launch.py`
3. 调用 `/spawn_entity` 把机器人模型塞进 Gazebo

URDF 不是直接读 `.urdf`，而是：
- 找到 `robot.xacro`
- 通过 `xacro.process_file(..., mappings=...)` 生成 URDF 字符串
- 再通过 `spawn_entity` 注入仿真

---

## 3. 传感器参数是如何传到模型里的（RGB 为例）

在 `race_gazebo.launch.py` 中：
- 定义了 `use_rgb_camera` 启动参数（默认 `false`）
- 并传入 xacro 映射：`'USE_RGB_CAMERA': use_rgb_camera`

在 `robot.xacro` 中：
- 定义 `xacro:arg name="USE_RGB_CAMERA"`
- include `gazebo.xacro`

在 `gazebo.xacro` 中：
- 用 `<xacro:if value="$(arg USE_RGB_CAMERA)">` 包裹 RGB 相机传感器块
- 插件 `libgazebo_ros_camera.so` 发布图像与相机信息话题

即：

`ros2 launch 参数 -> race_gazebo.launch.py mappings -> robot.xacro 参数 -> gazebo.xacro 条件块 -> Gazebo 传感器插件`

---

## 4. 当前脚本体系下的注意点（很关键）

1. `launchsim.py` 走的是 `launchgazebo.sh`，而它默认没有传 `use_rgb_camera:=true`。  
   也就是说：即便你已经写好 RGB 代码，走 `launchsim.py` 默认仍不会开 RGB。

2. 要启用 RGB，有两种方式：
   - 方式 A（推荐临时验证）：绕过 `launchsim.py`，直接手动 launch：

```bash
source /opt/ros/galactic/setup.bash
source /home/cyberdog_sim/install/setup.bash
ros2 launch cyberdog_gazebo race_gazebo.launch.py use_rgb_camera:=true paused:=false headless:=True
```

   - 方式 B（长期使用）：修改 `launchgazebo.sh`，在命令后加上 `use_rgb_camera:=true`。

3. 若出现 `gzserver exit code 255`，通常是已有旧 `gzserver/gzclient` 占用。先清理再启动：

```bash
pkill -f "ros2 launch cyberdog_gazebo race_gazebo.launch.py" || true
pkill -f gzserver || true
pkill -f gzclient || true
```

---

## 5. 验证链路是否成功

新开终端执行：

```bash
source /opt/ros/galactic/setup.bash
source /home/cyberdog_sim/install/setup.bash
ros2 topic list | grep -E "rgb|camera|image"
ros2 topic hz /rgb/image_raw
ros2 topic echo /rgb/camera_info --once
```

若 `topic hz` 有频率输出，说明 RGB 相机已经在仿真中正常工作。
